# HTTP-01 수동/자동 업로드 가이드 문구

## 1) 화면 목적
HTTP-01 검증에서 사용자가 토큰 파일을 올바르게 배치하도록 안내하고,
자동 업로드 또는 수동 업로드 경로를 명확히 선택하게 한다.

---

## 2) 공통 안내 문구
- "HTTP-01은 `http://<도메인>/.well-known/acme-challenge/<token>` 경로가 외부에서 접근 가능해야 합니다."
- "와일드카드(`*.example.com`)는 HTTP-01을 사용할 수 없습니다. DNS-01을 선택하세요."
- "검증 전 방화벽/리버스프록시/리다이렉트 설정을 확인하세요."

---

## 3) 자동 업로드 모드 문구

### 3.1 모드 설명
- 제목: **자동 업로드(권장)**
- 설명: "앱이 challenge 파일 생성 및 업로드를 시도합니다. 완료 후 접근 가능 여부를 자동 점검합니다."

### 3.2 입력 필드 라벨
- `웹루트 경로`: "웹서버가 정적 파일을 제공하는 루트 경로"
- `업로드 방식`: "로컬 파일쓰기 / FTP / SFTP"
- `연결 정보`: "호스트, 포트, 계정, 경로"

### 3.3 진행 상태 문구
1. "Challenge 파일 생성 중..."
2. "대상 경로 업로드 중..."
3. "외부 접근 확인 중..."
4. "검증 요청 준비 완료"

### 3.4 성공/실패 문구
- 성공: "파일 업로드 및 접근 확인이 완료되었습니다. ACME 검증을 진행합니다."
- 실패(업로드): "업로드에 실패했습니다. 연결 정보와 대상 경로를 확인해주세요."
- 실패(접근): "업로드는 되었지만 외부 접근 확인에 실패했습니다(HTTP {statusCode})."

---

## 4) 수동 업로드 모드 문구

### 4.1 모드 설명
- 제목: **수동 업로드**
- 설명: "아래 파일 내용을 웹서버 경로에 직접 업로드한 뒤, [접근 확인]을 실행하세요."

### 4.2 표시 정보
- `파일 경로`: `/.well-known/acme-challenge/<token>`
- `파일 내용`: `<keyAuthorization>`
- `검증 URL`: `http://<domain>/.well-known/acme-challenge/<token>`

### 4.3 사용자 액션 버튼 문구
- `파일 내용 복사`
- `경로 복사`
- `검증 URL 열기`
- `접근 확인`
- `검증 요청 진행`

### 4.4 체크리스트 문구
- "1) 파일을 정확한 경로에 업로드했습니다."
- "2) 파일 내용이 정확히 일치합니다(공백/개행 포함)."
- "3) 브라우저에서 검증 URL 접근 시 200 응답을 확인했습니다."

### 4.5 오류 안내 문구
- "404 Not Found: 파일 경로가 잘못되었거나 배포되지 않았습니다."
- "403 Forbidden: 웹서버 권한 또는 보안 정책을 확인하세요."
- "Timeout: 방화벽/네트워크 또는 도메인 DNS 설정을 확인하세요."
- "Unexpected Body: 파일 내용이 keyAuthorization과 일치하지 않습니다."

---

## 5) 고급 진단 문구
- "리다이렉트가 HTTPS 또는 다른 경로로 변경되면 검증에 실패할 수 있습니다."
- "CDN/캐시 사용 시 이전 파일이 응답될 수 있으니 캐시 무효화를 진행하세요."
- "WAF가 `/.well-known/acme-challenge` 경로를 차단하지 않는지 확인하세요."
