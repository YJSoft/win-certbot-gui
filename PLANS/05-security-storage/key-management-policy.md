# 키 관리 정책 문서

## 1) 목적
본 정책은 ACME 계정키/인증서 개인키의 생성, 저장, 사용, 폐기 전 과정을 정의한다.

## 2) 키 분류
- `ACME Account Key`: ACME 계정 식별 및 서명용 장기 키
- `Certificate Private Key`: 도메인 인증서 대응 키
- `Transient Key Material`: 메모리 내 일시적 처리 데이터

## 3) 생성 정책
- 키 생성은 Main/Core에서만 수행
- 권장 알고리즘/길이
  - RSA: 2048 이상
  - ECDSA: P-256 이상
- RNG는 OS CSPRNG 사용

## 4) 저장 정책
- 평문 저장 금지
- Windows DPAPI 기반 암호화 저장 우선
- 저장 경로 분리
  - `data/keys/` : 암호화된 키 블롭
  - `data/certificates/` : 공개 인증서/체인
- 키 메타데이터는 최소 정보만 보관(키 ID, 생성일, 용도)

## 5) 접근 정책
- Renderer 직접 접근 금지
- 키 사용 요청은 IPC를 통해 목적 기반 API로만 전달
- 복호화는 사용 시점 최소 범위에서 수행 후 즉시 메모리 정리

## 6) 회전/갱신 정책
- 인증서 갱신 시 키 재사용/재생성 정책을 사용자 옵션으로 제공
- 보안 사고 의심 시 즉시 키 폐기 및 신규 발급 유도
- 장기 미사용 키 정리 정책(예: 180일 미사용 시 검토)

## 7) 백업/복구
- 키 백업은 암호화된 형태로만 허용
- 복구 절차는 관리자 확인 단계 포함
- 복구 로그는 감사 대상으로 분리 저장

## 8) 폐기 정책
- 폐기 요청 시 키 참조 제거 + 저장 파일 안전 삭제(가능 범위 내)
- 폐기 이벤트를 audit 로그에 기록
- 폐기 후 복구 불가 경고를 사용자에게 명시
