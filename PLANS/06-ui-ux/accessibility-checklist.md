# 접근성 체크리스트 (A11y)

## 1) 키보드 접근성
- [ ] 모든 핵심 기능(발급/갱신/진단/내보내기)을 키보드만으로 수행 가능
- [ ] 탭 순서가 시각적 순서와 일치
- [ ] 모달 열림 시 포커스 트랩 및 닫힘 시 원위치 복귀
- [ ] Enter/Space 동작이 버튼/토글에 일관되게 적용

## 2) 스크린리더 호환
- [ ] 입력 필드에 명확한 `label` 연결
- [ ] 상태 배지/아이콘에 대체 텍스트 제공(`aria-label`)
- [ ] 진행 상태 변경(event.job.progress)은 `aria-live` 영역으로 안내
- [ ] 오류 메시지는 필드와 프로그램적으로 연결(`aria-describedby`)

## 3) 시각적 가독성
- [ ] 텍스트 대비(최소 WCAG AA) 충족
- [ ] 색상 외 수단(아이콘/텍스트)으로 상태 전달
- [ ] 폰트 확대(최대 200%) 시 기능 손실 없음
- [ ] 다크/라이트 테마 모두 대비 기준 충족

## 4) 인지 부담 최소화
- [ ] 단계형 마법사에서 현재 단계/전체 단계 표시
- [ ] 오류 발생 시 "문제 + 원인 + 다음 액션" 순서로 안내
- [ ] 기술 용어에 툴팁/도움말 링크 제공
- [ ] 장시간 대기(DNS 전파) 중 남은 예상 시간 또는 재시도 안내 제공

## 5) 폼/입력 검증
- [ ] 실시간 검증은 과도한 경고를 피하고 제출 시 요약 제공
- [ ] wildcard 도메인 입력 시 HTTP 비활성화 사유를 명확히 안내
- [ ] 실패 필드로 포커스 이동 및 오류 요약 제공
- [ ] 민감 입력(API 토큰)은 기본 마스킹 + 표시 토글 제공

## 6) 테스트 시나리오
- [ ] NVDA/내레이터로 발급 마법사 1회 완료 테스트
- [ ] 키보드만으로 HTTP-01 수동 업로드 플로우 수행 테스트
- [ ] 키보드만으로 DNS-01 수동 모드 전파 확인 플로우 테스트
- [ ] 고대비 모드에서 상태/오류 식별 가능 여부 테스트
