# 와이어프레임 (초안)

## 1) 대시보드 화면

```text
┌──────────────────────────────────────────────────────────────┐
│ WinCertbot Manager                          [환경: Staging] │
├──────────────────────────────────────────────────────────────┤
│ [새 인증서 발급] [갱신 실행] [설정] [로그/진단]              │
├──────────────────────────────────────────────────────────────┤
│ 인증서 목록                                                     │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 도메인            상태      만료일        남은일수      │ │
│ │ example.com       VALID     2026-08-10    D-99          │ │
│ │ *.example.com     EXPIRING  2026-05-30    D-27          │ │
│ └──────────────────────────────────────────────────────────┘ │
├──────────────────────────────────────────────────────────────┤
│ 최근 작업 로그                                                 │
│ - [성공] cert.issue.request job-123                           │
│ - [실패] DNS_PROPAGATION_TIMEOUT job-122                      │
└──────────────────────────────────────────────────────────────┘
```

## 2) 새 인증서 발급 마법사

### Step 1. 도메인 입력
```text
[CN 도메인] ______________________
[SAN 추가] _______________________
[+] SAN 추가

(안내) wildcard 포함 시 DNS-01만 사용할 수 있습니다.
[다음]
```

### Step 2. Challenge 방식 선택
```text
( ) HTTP-01
( ) DNS-01   (권장)

* wildcard 감지 시 HTTP-01 비활성화 + 이유 툴팁 노출
[이전] [다음]
```

### Step 3A. HTTP-01 설정
```text
모드: (자동 업로드) (수동 업로드)
웹루트 경로: ______________________
검증 URL 미리보기: http://domain/.well-known/acme-challenge/token

체크리스트
[ ] 파일 생성됨
[ ] 업로드 완료
[ ] 외부 접근 확인됨

[접근 확인] [검증 요청 진행]
```

### Step 3B. DNS-01 설정
```text
공급자: [Cloudflare v]
모드: (자동) (수동)
API 토큰: ******** (참조키)

TXT 이름: _acme-challenge.example.com
TXT 값:   xxxxx

상태
[ ] TXT 등록됨
[ ] 전파 확인됨
[ ] 검증 요청됨

[전파 확인] [검증 요청 진행]
```

### Step 4. 결과
```text
상태: 발급 성공 ✅
인증서 ID: cert-2026-0001
만료일: 2026-08-10
저장 위치: data/certificates/...

[PEM 내보내기] [대시보드로 이동]
```

## 3) 설정 화면
- Let's Encrypt 환경: staging / production (production 전환 확인 팝업)
- 기본 저장 경로
- 갱신 스케줄(자동/수동)
- 보안: 키 내보내기 허용 정책

## 4) 로그/진단 화면
- Job ID별 단계 로그 타임라인
- 실패 원인 카드(오류 코드 + 조치 가이드)
- HTTP 진단(URL/응답코드), DNS 진단(resolver별 결과)
